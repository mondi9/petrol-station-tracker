Security & Setup Guide

Google Maps API Integration

Petrol Station Tracker - Lagos Project

1. Create a Google Cloud Project

  1. Visit the Google Cloud Console.
  2. Click the Project Picker (top-left) and select New Project.
  3. Name it Fuel Tracker Lagos and click Create.

2. Enable Core Services

You must enable these three specific APIs for the app to function properly:

  • Distance Matrix API (Traffic-aware fuel station routing)
  • Geocoding API (Converting addresses to map coordinates)
  • Places API (High-quality location search)

3. Secure Your API Key

Important: Never share your API key. These restrictions ensure that even if someone sees your key, they cannot use it.
  1. Go to APIs & Services > Credentials.
  2. Click + CREATE CREDENTIALS and choose API Key.
  3. Click Edit Key to add restrictions:
    • Set Application restrictions to Websites.
    • Add http://localhost:* (for development).
    • Add https://petrol-station-tracker.netlify.app/* (for your live site).
  4. Set API restrictions to Restrict key and check the three APIs mentioned above.

4. Billing & Free Tier Verification

Google provides $200 of free monthly credit, which covers approx. 5,000–25,000 requests depending on the service.

Billing Setup: A linked card is required for identity verification. Google may request a temporary $10 authorization (it sits as a credit on your account).

5. Set "Never-Pay" Quotas

To ensure you never exceed the free monthly credit, set these daily limits:

  • Distance Matrix API: 150 requests per day.
  • Places/Geocoding API: 500 requests per day.

Settings found under: APIs & Services > Enabled APIs > [API Name] > Quotas & System limits.

6. Implementation

Once active, add the key to your .env.local file in the app root:

VITE_GOOGLE_MAPS_API_KEY=your_copied_key_here
